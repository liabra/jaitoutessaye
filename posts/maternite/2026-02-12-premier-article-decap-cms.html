Mon premier article avec Decap CMS

Découvrez comment j'ai intégré Decap CMS à mon blog sans toucher au design existant. Une solution parfaite pour gérer mes articles facilement.

👩‍💻
Par Marie Dev Full-Stack & Maman
<h2>🎉 Enfin un CMS qui respecte mon design !</h2> <p>Après avoir cherché pendant des semaines la solution parfaite pour gérer mes articles, j&#39;ai trouvé : <strong>Decap CMS</strong> (ex Netlify CMS).</p> <h3>Pourquoi j&#39;avais besoin d&#39;un CMS ?</h3> <p>Écrire des articles directement en HTML, c&#39;était :</p> <ul> <li>⏰ <strong>Chronophage</strong> : copier-coller le template à chaque fois</li> <li>🐛 <strong>Source d&#39;erreurs</strong> : oublier de fermer une balise</li> <li>📱 <strong>Pas mobile-friendly</strong> : impossible d&#39;écrire depuis mon téléphone</li> </ul> <h3>Ce que je cherchais</h3> <ol> <li><strong>Préserver mon design</strong> : Zéro modification du CSS/HTML existant</li> <li><strong>Workflow éditorial</strong> : Brouillons, révisions, publication</li> <li><strong>Interface intuitive</strong> : Markdown + preview</li> <li><strong>Sécurité</strong> : OAuth GitHub, pas de base de données</li> <li><strong>Gratuit</strong> : Open-source et sans frais cachés</li> </ol> <h2>✨ Decap CMS : la solution parfaite</h2> <h3>L&#39;installation</h3> <p>L&#39;intégration a pris moins d&#39;une heure :</p> <pre><code class="language-bash"># 1. Créer la structure mkdir -p admin content/posts/{maternite,tech} # 2. Configurer Decap CMS # admin/config.yml + admin/index.html # 3. Script de build node scripts/build.js </code></pre> <h3>Le workflow en pratique</h3> <ol> <li><strong>J&#39;écris</strong> dans l&#39;interface <code>/admin</code></li> <li><strong>Je prévisualise</strong> en temps réel</li> <li><strong>Je publie</strong> (commit GitHub automatique)</li> <li><strong>Railway rebuild</strong> et déploie automatiquement</li> </ol> <h3>Les avantages au quotidien</h3> <h4>✍️ Écriture facilitée</h4> <p>Je peux maintenant :</p> <ul> <li>Écrire depuis n&#39;importe quel appareil</li> <li>Utiliser le Markdown (bien plus rapide que HTML)</li> <li>Avoir une preview en direct</li> <li>Organiser mes brouillons</li> </ul> <h4>🎨 Design préservé</h4> <p>Le CMS génère du HTML en utilisant <strong>exactement</strong> mon template existant :</p> <ul> <li>Mêmes classes CSS</li> <li>Même structure DOM</li> <li>Mêmes scripts JS</li> <li>Même esthétique</li> </ul> <h4>🚀 Workflow professionnel</h4> <pre><code>Brouillon → Révision → Publication </code></pre> <p>Fini les articles publiés par erreur avec des fautes !</p> <h2>💻 Détails techniques</h2> <h3>Architecture</h3> <pre><code>content/posts/maternite/*.md → [Build Script] → posts/maternite/*.html ↓ index.html (updated) </code></pre> <h3>Le build script</h3> <p>Le script <code>build.js</code> :</p> <ol> <li>Parse le frontmatter YAML</li> <li>Convertit le Markdown en HTML</li> <li>Injecte dans le template</li> <li>Met à jour <code>index.html</code></li> </ol> <h3>Configuration GitHub OAuth</h3> <pre><code class="language-yaml">backend: name: github repo: liabra/jaitoutessaye branch: main base_url: https://oauth-worker.workers.dev </code></pre> <h2>🎯 Résultat</h2> <h3>Avant Decap CMS</h3> <ul> <li>⏱️ <strong>30 min</strong> pour créer un article</li> <li>🐛 <strong>Erreurs HTML</strong> fréquentes</li> <li>📱 <strong>Pas de mobile</strong></li> <li>😰 <strong>Stress</strong> à chaque publication</li> </ul> <h3>Après Decap CMS</h3> <ul> <li>⏱️ <strong>10 min</strong> pour créer un article</li> <li>✅ <strong>Zéro erreur</strong> (Markdown + validation)</li> <li>📱 <strong>Écriture mobile</strong> possible</li> <li>😌 <strong>Workflow serein</strong></li> </ul> <h2>🌸 Pour conclure</h2> <p>Decap CMS a transformé ma façon de bloguer. Je peux maintenant :</p> <ul> <li>Me concentrer sur l&#39;<strong>écriture</strong></li> <li>Publier depuis mon <strong>téléphone</strong></li> <li>Gérer un <strong>workflow professionnel</strong></li> <li>Garder mon <strong>design intact</strong></li> </ul> <p>Si vous avez un blog statique et que vous hésitez à ajouter un CMS, <strong>foncez</strong> ! C&#39;est exactement ce qu&#39;il vous faut.</p> <hr> <p><strong>Prochaine étape</strong> : Intégrer un système de commentaires privacy-first (Isso ou Commento).</p> <p><em>Avez-vous déjà utilisé Decap CMS ? Partagez votre expérience en commentaire !</em></p>
